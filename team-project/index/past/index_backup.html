파일: functions/longitudinal_controller_fcn.m

입력현재 Source상태

⚙️ 알고리즘 상세

1. Werling Frenet Optimal Trajectory (Dev-C)

차선 변경 trajectory를 Quintic(횡) + Quartic(종) 다중 후보로 생성, 비용 최소 선택.

%% Werling cost function
c_lat = K_J·∫ḋ³² + K_T·T + K_D·(d_f − target_d)²
c_lon = K_J·∫s⃛² + K_T·T + K_V·(v_target − s_f)²
c_tot = K_LAT·c_lat + K_LON·c_lon + K_TTC/ttc(lane)

%% 제약 (위반 시 reject)
max|v| ≤ V_MAX_FRENET (30 m/s)
max|a| ≤ ACC_MAX_FRENET (4 m/s²)
max|κ| ≤ K_MAX_CURV (0.20 1/m)

2. lane_cost 차선 선택 (Dev-D)

각 차선의 위험/이점을 비용 함수로 평가, argmin으로 자동 선택.

lane_cost(i) = K_BLOCK·blocked(i)
             + K_TTC/max(ttc(i), 0.5)
             + K_GAP/max(front_gap(i), 5.0)
             + K_LATBIAS·|i - current_lane|

[~, target_lane] = min(lane_cost);    %% argmin

3. Mode State Machine (Dev-B)

4. PID + ACC (Dev-A 종방향)

e_v = v_target - v_ego              %% 속도 오차
e_d = front_gap - safe_dist         %% 거리 오차 (ACC)
e_eff = min(e_d, e_v)               %% 더 보수적 선택

acc_cmd = Kp·e_eff + Ki·∫e + Kd·de/dt   %% PID
if front_gap < 5: acc_cmd = -5           %% 비상 제동
acc_cmd = clip(acc_cmd, -5, +3)          %% 한계

5. Lateral PID (Dev-E)

e = -cte                            %% DevDist 양수 → 좌조향(+)

⚠️ 트러블슈팅

셋업 과정에서 마주친 4가지 함정 + 해결법 (이미 해결됨)

함정 1

시간 값은 감소하지 않아야 합니다

증상

From Workspace에서 workspace 입력으로 유효하지 않은 작업 공간 변수
시간 값은 감소하지 않아야 합니다.

원인

base workspace의 waypoints가 [x, y] 877×2 형식. From Workspace 블록은 첫 column을 시간으로 해석. 도로 곡선부에서 x좌표가 감소하므로 시간 감소로 인식.

해결

setup_paths.m이 자동으로 [time, x, y] 877×3 형식으로 변환. From Workspace 블록은 Do Not Modify 영역이라 그대로 둠.

함정 2

BusSelector 입력이 Bus 신호가 아닙니다

증상

Bus Selector 블록에 대한 입력은 가상 버스 신호 또는
비가상 버스 신호여야 합니다.

원인

BusCreator3 → Goto[Ego_Global_Pos] → From → BusSelector 경로에서 Bus type 정보 손실.

해결

BusSelector 제거. Modify 영역에 신규 Read CM Dict 3개 추가:

  • Read_Ego_X_Lib → Goto[Ego_Global_X]
  • Read_Ego_Y_Lib → Goto[Ego_Global_Y]
  • Read_Ego_Yaw_Lib → Goto[Ego_Yaw]
함정 3

Goto/From이 atomic 경계 못 넘음

증상

Goto/From connections cannot cross nonvirtual subsystem boundaries.

원인

Lib_*는 Function-Call Triggered Subsystem = 강제 nonvirtual atomic. 안과 밖 신호는 Inport/Outport로만 전달 가능.

해결

Lib 안의 From/Constant 모두 제거. Lib에 Inport/Outport 추가. 외부에서 From/Constant → Lib.Inport, Lib.Outport → Goto.

Lib 안:  Inport_1 → Wrapper → Outport
Lib 밖:  From/Constant → Lib.Inport,  Lib.Outport → Goto
함정 4

function-call signal 단순 분기 불가

증상

function-call signal is branched incorrectly.
needs to be recreated using a Function-Call Split block.

원인

같은 트리거(do_Logic_10ms_1)를 2개 Lib(Long, Lat)가 단순 line branch로 받음.

해결

FCS_10ms_LongLat Function-Call Split 블록 추가 (1 in → 2 out).

함정 5

Day6 시나리오 활성화로 변수 못 찾음

증상

Variable 'Traffic.T01.rzv' not in dictionary

원인

day7_final TestRun에는 Traffic.T22~T28만 있는데, Day6/Scenario 1 서브시스템이 동시에 활성이라 Traffic.T01.rzv 변수를 찾으려 함.

해결

Day3/Scenario 1, Day6/Scenario 1 commented out. Final Project만 활성. 학습용 Day 시뮬 원하면 Setup 페이지의 토글 코드 사용.
